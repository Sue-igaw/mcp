# 모바일인덱스 MCP 사업 기획서 v2.1

> 데이터를 보여주는 것을 넘어,
> **지금 필요한 답을 만들어내는 새로운 시작**

---

## 01 문제 인식

### 🤔 데이터는 많지만, 제대로 활용되지 않는다

모바일인덱스는 국내 대표 모바일 앱 데이터 플랫폼이지만, 많은 고객이 데이터를 조회하는 수준에 머무르고 있습니다.

| 문제 | 설명 |
|------|------|
| 📊 단순 조회 중심 | 월 300만 원을 내고 MAU 숫자 확인 수준에 머무름 |
| 📅 사용 편중 | 연간 계약이지만 보고서 시즌에만 집중 사용 |
| ❓ ROI 불명확 | 광고비는 ROAS로 증명 가능하지만, 데이터 구독은 성과 연결이 어려움 |

> 이것은 고객의 문제가 아니라, **데이터 활용 구조의 한계입니다**

### 🤯 데이터를 활용하는 과정이 지나치게 복잡하다

수집 + 정리에 전체 작업 시간의 60~70%가 소모됩니다. 분석과 의사결정까지 도달하기 어렵습니다.

| ❌ 현재 구조 | ✅ 필요한 구조 |
|-------------|--------------|
| 수집/정리에 전체 시간의 60~70% | 질문 → 바로 활용 가능한 결과 |
| 도구 4개를 왔다갔다 | 수집/정리 과정 제거 |
| 경쟁앱 5개 분석 = 같은 조작 5번 반복 | 분석과 판단에 집중 |

### 📊 고객이 원하는 것은 데이터가 아니라 "바로 활용 가능한 결과"

| ❌ BEFORE | ✅ AFTER |
|-----------|----------|
| 데이터 제공 → 고객이 해석 | **데이터 기반 답 생성 → 바로 활용** |
| 과정 중심 | **결과 중심** |
| 활용 어려움 | **즉시 활용 가능** |

> 데이터의 가치는 설명이 아니라 **사용 경험으로 증명됩니다**

### 🤔 왜 데이터를 안 쓰는가?

필요 없는 것이 아니라, **"이 데이터가 정말 필요한지 확신이 없다"**

```
확신 없음  →  사용 안 함  →  가치 체감 없음  →  계속 안 씀
```

> 사용하지 않기 때문에 확신이 없고, **확신이 없기 때문에 구매하지 않는다**

## 02 시장 변화

### 🧑‍💻 데이터 활용 방식이 바뀌고 있다

사용자는 더 이상 데이터를 찾지 않습니다. **결과를 바로 얻고 싶어합니다.**

- **검색 → 질문** — 키워드를 입력하는 것이 아니라, 자연어로 질문합니다
- **조회 → 결과** — 대시보드를 탐색하는 것이 아니라, 바로 답을 받습니다
- **도구 사용 → AI 활용** — 여러 도구를 오가는 것이 아니라, AI 하나로 해결합니다

| 지표 | 수치 |
|------|------|
| 🤖 한국 직장인 AI 사용 경험 | 63.5% |
| 💼 업무 목적 AI 사용률 | 51.8% |
| 🇰🇷 한국 업무용 AI 도입률 | 미국(26.5%)의 약 2배 |
| ⚔️ 경쟁사 대응 | 시밀러웹 2025.09 MCP 정식 출시 |

*출처: 한국은행 "AI의 빠른 확산과 생산성 효과" 보고서, 2025.08*

> 모바일 앱 심화 데이터를 MCP로 제공하는 서비스는 **아직 시장에 없습니다**

## 💡 PIVOT

### 데이터를 파는 것에서, "답을 만드는 것"으로

데이터는 더 이상 최종 결과가 아닙니다. 데이터는 중간 과정이고, 사용자는 지금 필요한 답을 원합니다.

## 03 해결 방법

### 💡 MCP는 데이터를 "써보게" 만드는 구조다

설명이 아니라 경험을 통해 데이터의 가치를 느끼게 합니다.
MCP는 데이터를 더 잘 보여주는 도구가 아니라, **데이터로 답을 만들어내는 구조**입니다.

### 🔥 같은 질문, 완전히 다른 결과

"GPT한테 물어보면 되는데, 왜 MCP가 필요해?" — 가장 많이 받는 질문입니다.

| ChatGPT | 모바일인덱스 MCP |
|---------|-----------------|
| 이미 존재하는 정보/리포트 기반 | **실제 모바일인덱스 데이터 기반** |
| 과거 데이터 재사용 | **현재 기준으로 재생성** |
| 설명 중심 | **분석 결과 중심** |
| 참고용 | **의사결정 활용 가능** |

> **핵심 메시지: GPT는 답을 가져오고, MCP는 지금 필요한 답을 만들어냅니다**

GPT는 이미 만들어진 리포트나 콘텐츠를 기반으로 답을 제공합니다.
MCP는 실제 데이터를 기반으로, **조건과 시점에 맞는 답을 새롭게 생성**합니다.

> 리포트는 과거를 설명하고, **MCP는 현재를 분석합니다**

### 🚨 구체적으로 뭐가 다른가

| 질문 | GPT 단독 | 모바일인덱스 MCP |
|------|---------|-----------------|
| "카카오톡 MAU?" | "약 4,700만 명으로 추정" ⚠️ 출처 없음, 검증 불가 | "2026년 3월 기준 4,612만 명" ✅ 실측, 시점 명시 |
| "쿠팡이츠 vs 배민 추이" | "배민이 1위이며 성장 중" ⚠️ 정성적, 수치 없음 | 월별 MAU 추이 + 증감률 + 교차 분석 ✅ 수치 기반 |
| "증권 업종 TOP 10" | "키움증권, 토스증권 등" ⚠️ 근거 없음 | 이번 주 기준 순위 + 전주 대비 변동 ✅ 실시간 |

### 📌 GPT가 구조적으로 못하는 3가지

1. **실시간 데이터** — GPT는 학습 시점 이후의 데이터를 모릅니다. 웹 검색을 해도 앱 사용량 데이터는 공개되어 있지 않습니다.
2. **비공개 심화 데이터** — 성별/연령별 사용자 구성, 이탈률, 동시 사용 앱, 페르소나 같은 데이터는 인터넷에 없습니다.
3. **일관된 비교 분석** — 동일한 기준, 동일한 시점의 데이터가 있어야 가능합니다. GPT는 출처가 다른 조각 정보를 짜깁기합니다.

### 💡 그래서 MCP는 뭔가?

MCP는 AI가 아닙니다. **AI가 모바일인덱스 데이터를 직접 조회할 수 있게 연결하는 통로**입니다.

```
GPT 단독:   사용자 질문 → GPT가 추정해서 답변 (출처 불명, 검증 불가)
GPT + MCP:  사용자 질문 → GPT가 모바일인덱스 실제 데이터를 조회 → 데이터 기반 답변
```

GPT의 대체재가 아니라 **GPT를 더 똑똑하게 만드는 도구**입니다.

### 🎯 모바일인덱스 MCP의 포지셔닝

> **"데이터를 찾는 도구가 아니라, 데이터로 답을 만드는 구조"**

### 👥 타겟별로 MCP가 의미하는 것

| 타겟 | MCP가 주는 가치 | 키 메시지 |
|------|----------------|----------|
| MI 기존 고객 | 대시보드 조작 없이 AI로 바로 분석 | **"매번 클릭하던 그 작업, 이제 한 마디면 됩니다"** |
| MI를 아는데 안 쓰는 사람 | 가입/학습 없이 AI에서 바로 체험 | **"가입 전에 먼저 써보세요. AI에서 바로 됩니다"** |
| MI를 모르는 사람 | AI가 만든 결과물로 모바일인덱스 인지 | **"이 보고서, AI한테 한 마디 했더니 3분 만에 나왔습니다"** |

> MCP는 모바일인덱스를 **"써보게 만드는 도구"**이자, **"데이터로 답을 만들어내는 경험 설계"**입니다


### 🤷 MCP 상품 출시 VS 자체 AI 기능 탑재

| 항목 | A. MCP 상품 출시 | B. 자체 AI 기능 탑재 |
|------|------------------|----------------------|
| 개발 범위 | MCP 서버 (중간 레이어) | AI 모델 연동 + 채팅 UI + 프롬프트 설계 |
| 개발 기간 | **6~8주** | 12~16주 이상 |
| 도달 범위 | **Claude/ChatGPT/Gemini 전체 사용자** | 모바일인덱스 기존 고객만 |
| AI 품질 | Claude, GPT 등 최고 수준 AI 활용 | 자체 구현 수준에 의존 |
| UI/UX 제어 | AI 플랫폼에 의존 (제한적) | **완전 제어 가능** |
| 비용 | **서버 운영비만** | AI API 호출 비용 + 개발비 |
| 검증 속도 | **빠르게 시장 반응 확인 가능** | 출시까지 시간이 걸려 검증이 늦음 |

> MCP는 확산을 위한 전략이고, 자체 AI는 완성을 위한 전략입니다.

### 🎬 MCP로 시작, 모바일인덱스 자체 AI로 완성

| Phase 1 — 지금 | Phase 2 — 검증 후 | Phase 3 — 중장기 |
|----------------|-------------------|------------------|
| MCP 베타 출시 | MCP 정식 출시 + 자체 AI 설계 착수 | 자체 AI 출시 + 전사 MCP 확장 |
| 사용 데이터 수집, 시장 반응 검증 | 사용 패턴 기반 자체 AI UI/시나리오 설계 | 모바일인덱스 내 AI 인터페이스 |

## 04 무엇이 바뀌는가

### 😮 질문만 하면, 답이 만들어진다

| 💬 질문 | 🤖 MCP 결과 |
|--------|------------|
| "우리 앱 MAU가 떨어지고 있어" | 경쟁앱 MAU 추이 비교 → 이탈 분석 → 원인 진단 |
| "신규 앱 출시를 준비 중이야" | 타겟 업종 시장 규모 → 경쟁 구도 → 포지셔닝 제안 |
| "마케팅 예산을 어디에 써야 할지 모르겠어" | 데모그래픽 분석 → 경쟁앱 사용자 비교 → 타겟 세그먼트 추천 |
| "경영진 보고가 내일이야" | 시장 현황 → 자사 포지션 → 경쟁사 동향 → 보고서 자동 생성 |

> 사용자는 데이터를 보는 것이 아니라, **답을 바로 활용합니다**

### 🙌 "복붙 AI"와 "연결된 AI"는 다르다

| 항목 | 일반 AI 활용 (복붙) | MCP (연결) |
|------|-------------------|-----------|
| 데이터 수집 | 직접 복사해서 붙여넣기 | **자동 호출** |
| 최신성 | 복사 시점에 고정 | **실시간 최신 데이터** |
| 반복 작업 | 매번 다시 복사 필요 | **자동화 가능** |
| 분석 | 수동 정리 후 질문 | **질문만 하면 자동 분석** |

> MCP는 데이터를 전달하는 것이 아니라, **데이터를 활용 가능한 상태로 만듭니다**

### ⚠️ MCP의 역할 (중요)

MCP는 결과물을 직접 만드는 엔진이 아닙니다.
AI가 답을 만들고, MCP는 데이터를 연결합니다.

> **핵심 역할: 데이터를 AI가 바로 사용할 수 있게 만드는 것**
