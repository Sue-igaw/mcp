# <1> 프로젝트 배경

## 1. MCP란 무엇인가

💡 MCP(Model Context Protocol)는 AI가 외부 서비스의 데이터를 직접 가져다 쓸 수 있게 해주는 표준 프로토콜 <br>
💡 MCP 서버 하나를 만들면 Claude, ChatGPT, Gemini, Copilot, Kiro 등 모든 MCP 지원 AI 플랫폼에서 사용 가능

- 2024년 11월 Anthropic(Claude 개발사)이 오픈소스로 발표
- 2025년 12월 Linux Foundation 산하 Agentic AI Foundation(AAIF)에 기증 → 업계 공통 표준으로 전환
- Anthropic, OpenAI, Google, Microsoft, AWS가 공동 참여
- 2026년 3월 기준 10,000개 이상의 MCP 서버 운영 중, SDK 월간 다운로드 9,700만 건 이상

<br>

## 2. MCP 도입 사례 — 시밀러웹

💡 시밀러웹(SimilarWeb)은 2025년 9월 MCP 서버 공식 출시, 자사 웹/앱 분석 데이터를 AI 플랫폼에서 직접 조회할 수 있도록 제공 중 <br>
💡 시밀러웹은 웹 트레픽 중심이며, 앱 분석은 3개 도구에 불과 <br>
💡 모바일 앱 심화 데이터(사용량, 이탈, 페르소나 등)를 MCP로 제공하는 서비스는 시장에 없는 것으로 파악, 선점 기회 존재 <br>

**시밀러웹 MCP 제공 방식**
- MCP를 별도 상품으로 판매하지 않고, 기존 API 구독(Business, Enterprise 등)에 MCP 접근을 포함
- MCP 사용 시 기존 API와 동일한 Data Credits이 차감되는 구조
- 엔드포인트: `https://mcp.similarweb.com`
- Claude, ChatGPT, Cursor, Copilot, Zapier, n8n 등과 연동 가능

**시밀러웹 MCP 도구 구성**
- 웹사이트 분석: 18개 도구 (트래픽, 유입 채널, 순위, 인구통계, 광고비 등)
- 검색 분석: 2개 도구 (SERP 순위, 클릭 추이)
- 앱 분석: 3개 도구 (설치 비율, 앱스토어 순위, 앱 기본 정보)

## 3. 왜 모바일인덱스에 MCP가 필요한가

### 3-1. 시장 흐름

- 디지털 인터페이스가 웹/앱 대시보드 → AI 대화형 인터페이스로 전환 중
- 사용자가 직접 대시보드를 조작하는 대신, AI에게 자연어로 요청하는 방식으로 변화
- 경쟁사 시밀러웹(SimilarWeb)이 2025년 9월 MCP를 이미 출시

### 3-2. 모바일인덱스 데이터의 MCP 적합성

- 이미 적재 · 가공된 통계 데이터 → 별도 데이터 파이프라인 없이 기존 API로 바로 제공 가능
- 정형화된 수치 데이터 → AI가 잘못 해석할 가능성이 낮음 (할루시네이션 리스크 낮음)
- 기존 Insight API가 이미 운영 중 → MCP 서버는 이 API를 AI에 연결하는 중간 레이어만 추가하면 됨

### 3-3. AI에 데이터를 제공 vs 서비스에 AI를 탑재

💡 개발 부담이 적고, 도달 범위가 넓은 MCP 상품 우선 출시가 적합 <br>
💡 자체 AI 기능은 MCP 출시 이후 중장기 과제로 검토 가능

| 항목 | MCP (AI에 데이터를 제공) | 자체 AI 기능 (서비스에 AI를 탑재) |
|------|----------------------|-------------------------------|
| 개념 | 외부 AI(Claude, ChatGPT 등)가 우리 데이터를 가져감 | 모바일인덱스 웹사이트 안에 AI 챗봇/분석 기능 추가 |
| 사용 환경 | 사용자가 쓰는 AI 플랫폼 | 모바일인덱스 웹사이트 |
| 개발 범위 | MCP 서버 (상대적으로 가벼움) | AI 모델 연동 + 채팅 UI + 프롬프트 설계 등 |
| 개발 기간 | 6~8주 | 12~16주 이상 |
| 도달 범위 | Claude/ChatGPT/Gemini 전체 사용자 | 모바일인덱스 기존 고객만 |
| AI 품질 | Claude, GPT 등 최고 수준 AI 활용 | 자체 구현 수준에 의존 |
| 비용 | 서버 운영비만 | AI API 호출 비용 + 개발비 |
| 경쟁사 대응 | 시밀러웹이 이미 MCP 출시 | 시밀러웹도 자체 AI 에이전트 개발 중 |

## 4. 활용 시나리오

**모바일인덱스** — 앱 시장 데이터 분석
```
사용자: 카카오톡과 네이버 앱의 최근 6개월 MAU 추이를 비교 분석해서, 마케팅 전략 보고서를 작성해줘
AI: MCP를 통해 모바일인덱스 데이터 자동 조회, 데이터 분석 + 리포트 작성 + 전략 제안
```

**디파이너리** — 자사 앱 고객 분석 + 캠페인 최적화
```
사용자: 지난달 푸시 캠페인 A/B 테스트 결과를 비교하고, 이탈 위험 사용자 세그먼트를 뽑아줘
AI: MCP를 통해 디파이너리 캠페인 성과 데이터 조회, A/B 비교 분석 + 이탈 위험 세그먼트 추출 + 개선안 제안
```

**트레이딩웍스** — 광고 성과 분석 + 예산 최적화
```
사용자: 이번 주 캠페인별 ROAS를 비교하고, CTR이 낮은 광고 소재를 찾아서 예산 재배분 안을 만들어줘
AI: MCP를 통해 트레이딩웍스 광고 집행 데이터 조회, 캠페인별 성과 비교 + 저성과 소재 식별 + 예산 최적화 제안
```

  3️⃣ AI 플랫폼 전환 비용이 0이므로 구매 결정이 상대적으로 가벼움

## 2. 과금 모델

### 2-1. 과금 구조

💡 기본적인 과금 구조는 **사용자에게 API 키를 발급하고, 해당 API 키를 통해 인증 및 과금을 제어하는 방식** <br>
💡 사용자는 AI 플랫폼 사용료와 별도로, 모바일인덱스 MCP 데이터 사용료를 IGAW에 직접 지불하는 방식

```
① 사용자가 모바일인덱스 사이트에서 요금제 선택 & 결제
② API 키 발급
③ 사용자가 AI 플랫폼(ex. GPT, Claude, ...)에 API 키 입력
④ MCP 서버가 API 키로 인증 → 요금제에 따라 데이터 제공
```

### 2-2. 요금제

**💡 요금제는 크게 2가지 방향 존재** <br>

    1️⃣ 월정액 (선불) — 월 단위로 미리 결제, 구독 기간 동안 자유롭게 사용
    2️⃣ 사용량 기반 (후불) — 실제 API 호출량에 따라 과금
  
<br>

**1️⃣ 월정액 (선불)** | 사용량 제어 없이, 구독 기간 동안 자유롭게 사용 가능 <br>

📌 MCP 기능별(ex. ESSENTIAL, PREMIUM, ...)로 상품 분리 가능

|--------|--------|---------------------|-----------|----------|
| Free | 무료 | - | 월 20회 (체험용) | 앱 검색, 업종 분류, Top 10 순위만 |

| Enterprise | 별도 협의 | 별도 협의 | 무제한 | 전체 + 커스텀 + 전용 서버 + SLA |

**2️⃣ 사용량 기반 (후불)** | 실제 API 호출량에 따라 과금되는 후불제 <br>

📌 현재 모바일인덱스 API 구조는 1회 호출 = 1개 데이터 요청으로, API 호출 횟수 = 사용량 <br>
📌 호출 데이터 유형(ex. 사용자수, 유입/이탈, ...)에 따라 호출 단가 다르게 설정 가능
  

| 구분 | 내용 |
|------|------|
| 기본료 | 월 기본료 없음 (또는 최소 기본료 설정 가능) |
| 호출당 단가 | 호출 1회당 n원 (데이터 종류별 차등 가능) |
| 정산 | 월말 정산, 익월 청구 |
| 상한선 | 월 상한 금액 설정 가능 (예: 300만원 초과 시 자동 차단) |

**3️⃣ 기존 API 구독에 포함 (시밀러웹 방식)** | MCP를 별도 상품으로 팔지 않고, 기존 API 구독에 MCP 접근을 포함시키는 방식 <br>

📌 시밀러웹이 이 방식을 채택 — API 구독 고객이 동일한 Data Credits으로 MCP도 사용 <br>
📌 MCP 단독 매출은 없지만, 기존 구독 상품의 가치를 높여 이탈 방지 + 신규 가입 유도 효과

| 구분 | 내용 |
|------|------|
| MCP 별도 가격 | 없음 (기존 API 구독에 포함) |
| 과금 방식 | 기존 API와 동일한 크레딧/호출 횟수 차감 |
| 장점 | 기존 고객 즉시 사용 가능, 추가 결제 없이 MCP 체험 |
| 단점 | MCP에서 직접적인 추가 매출 발생하지 않음 |

**[참고 1] 1안 vs 2안 vs 3안 비교** <br>

💡 1안 (월정액)으로 출시 이후, 사용 패턴 데이터가 쌓이면 2안 도입 검토하는 방향 권장 (사용자 입장에서도 단순하고, 운영 측면에서도 결제 및 정산 간단) <br>
💡 2안으로 진행 시, 정산 및 청구 시스템(사용량에 따라 후불 결제) 추가 개발 필요 <br>
💡 3안은 시밀러웹 방식으로, MCP 단독 매출보다 기존 구독 가치 강화에 초점

| 항목 | 1안 (월정액) | 2안 (사용량 후불) | 3안 (기존 API 구독에 포함) |
|------|------------|------------|------------------------|
| 사용자 예측 가능성 | ✅ 매달 같은 금액 | ❌ 얼마 나올지 모름 | ✅ 기존 구독료에 포함 |
| 헤비 사용자 | 유리 (무제한) | 불리 (많이 쓰면 비쌈) | 유리 (기존 크레딧 내 사용) |
| 라이트 사용자 | 불리 (안 써도 같은 금액) | 유리 (쓴 만큼만) | 유리 (추가 비용 없음) |
| 운영 복잡도 | 낮음 | 높음 (정산/청구 시스템 추가 개발 필요) | 낮음 (기존 시스템 활용) |
| 매출 예측 | 쉬움 (구독자 수 × 단가) | 어려움 (사용량 변동) | MCP 단독 매출 없음 |
| 신규 매출 | ✅ MCP 별도 매출 | ✅ MCP 별도 매출 | ❌ 추가 매출 없음 (기존 구독 가치 강화) |
| 참고 사례 | - | 시밀러웹 Data Credits | 시밀러웹 (API 구독에 MCP 포함) |

| 요금제 변경/해지 | 업그레이드, 다운그레이드, 구독 취소 |

> 📌 기존 모바일인덱스 구독 과금 시스템이 있다면, "MCP 요금제"를 추가하는 게 가장 빠르다.

개발 소요: 3~4주 (기존 시스템 활용 시 2주)

## 6. 전체 개발 로드맵

```
Phase 1: MCP 서버 정식화
├── 클라우드 배포
├── 인증/과금 미들웨어
├── 에러 처리 & 로깅
└── 내부 QA

Phase 2: 상품 사이트
├── 랜딩 페이지 (홍보)
├── 가격 페이지
├── 가입/결제 플로우
├── 마이페이지 (API 키, 사용량)
└── 연결 가이드 문서

Phase 3: 베타 테스트
├── 내부 테스트
├── 소수 고객 베타 (5~10곳)
└── 피드백 반영

Phase 4: 정식 출시
├── 홈페이지 공개
├── 기존 고객 안내
└── 마케팅 시작
```

# <4> 의사결정 필요 사항

## 1. 상품 체계
- [ ] MCP 기능 범위: 1차 출시 시 전체 27개 도구 vs 핵심만 먼저 / 기능별 패키지 분리 여부
- [ ] Free 제공 범위: 무료 도구 목록, 호출 횟수(월 20~100회), 조회 기간 제한
- [ ] 가격 체계: 1안(월정액) vs 2안(후불) 선택, Standard·Pro 가격 확정, 연간 할인율, 기존 구독 고객 정책

## 2. 결제
- [ ] 결제 수단: 카드 결제(PG사 연동) 포함 여부, 세금계산서 발행
- [ ] 결제 시스템: 기존 모바일인덱스 시스템 활용 vs 신규 구축

## 3. 플랫폼 및 배포
- [ ] 1차 지원 AI 플랫폼 범위 (Claude + ChatGPT 먼저?)
- [ ] 마켓플레이스 등록 여부 및 우선순위
- [ ] 상품 사이트: 기존 사이트 확장 vs 별도 서브도메인

## 4. 운영 정책
- [ ] API 키 공유 방지 및 동시 접속 제한 정책
- [ ] 비정상 트래픽 기준 정의
- [ ] 베타 테스트 대상·기간·피드백 수집 방법
- [ ] 정식 출시 목표 시점
