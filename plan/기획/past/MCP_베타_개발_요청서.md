# 모바일인덱스 MCP 베타 — 개발 협업 요청서

<br>

## 1. 개요

### 1-1. 배경

MCP 상품 기획 방향에 따라 기존 고객 대상 베타 테스트를 먼저 진행하고자 합니다.

AM 팀 의견을 반영한 기대효과는 다음과 같습니다.

- 단기 매출 효과는 미미할 것으로 예상
  - 모바일인덱스 데이터 활용이 MAU 쏠림이 심하며, MCP 활용을 위해서는 MAU 외 다양한 데이터 활용이 필요
  - 리서치/대행사/연구원 등을 제외하면 경쟁앱 데이터만 확인하는 경우가 많고, 경쟁앱이 적게는 2개, 많아도 20개 이내 수준
- 단, 프리미엄 고객 한정 기능으로 제공 시 **에센셜 고객의 버전업** 또는 **기존 고객 락인 효과** 기대
- 베타 고객사 섭외는 내부 보안 이슈가 있는 곳만 아니라면 어렵지 않을 것으로 판단

### 1-2. 테스트 현황

AI 플랫폼별로 로컬 MCP 서버를 연결하여 동작을 검증했습니다.

| AI 플랫폼 | 연결 결과 | 베타 포함 | 설명 |
|----------|----------|----------|------|
| Kiro | ✅ 성공 | ✅ 포함 | AWS AI IDE. MCP 네이티브 지원 |
| Claude Desktop | ✅ 성공 | ✅ 포함 | Anthropic 데스크탑 앱. MCP 최초 개발사 |
| ChatGPT | 미테스트 | ✅ 포함 | 원격 서버 배포 후 테스트 예정. 공식 문서상 Plus 이상에서 MCP 지원하나, 현재 커스텀 MCP 서버 연결 UI는 Team/Business/Enterprise 위주로 점진 배포 중. Plus에서는 아직 메뉴가 안 열릴 수 있음 |
| Cursor | 미테스트 | ✅ 포함 | 원격 서버 배포 후 테스트 예정. AI 코드 에디터. 무료 플랜도 MCP 지원 (사용량 제한 있음) |
| Windsurf | 미테스트 | ✅ 포함 | 원격 서버 배포 후 테스트 예정. AI 코드 에디터. 무료 플랜도 MCP 지원 (사용량 제한 있음) |
| Gemini | 미테스트 | ❌ 제외 | CLI(개발자 터미널) 환경에서만 MCP 지원. 일반 웹/앱 미지원 |

사용하는 LLM 및 에이전트에 따라 답변 표현이 다를 수 있으나, 데이터 조회 결과 자체는 동일하며 큰 차이는 없습니다.

### 1-3. 목표

모바일인덱스 MCP 서버를 원격(클라우드)에 배포하여,
베타 사용자가 **API 키 + 서버 URL만으로** AI 플랫폼에 연결할 수 있도록 합니다.

사용자에게는 **연동 가이드 웹페이지**와 **개인 API 키**를 전달하며,
가이드에 따라 설정하면 AI에서 모바일인덱스 데이터를 자연어로 조회할 수 있습니다.

**베타 대상**
- 기존 고객사 중 베타 참여 의향이 있는 곳 (AM 팀 섭외)
- 내부 보안 이슈가 없는 고객사 우선
- 현재 AI 플랫폼(Claude, ChatGPT, Kiro 등)을 사용 중인 담당자가 있는 곳 (바로 실행 가능)

**제공 범위**
- MCP 서버 전체 27개 도구 (Insight API 전체 기능)
- 지원 AI 플랫폼: Kiro, Claude Desktop, ChatGPT, Cursor, Windsurf

> 참고: Gemini도 MCP를 지원하지만, 현재는 Gemini CLI(개발자 터미널 도구) 환경에서만 가능합니다. 일반 Gemini 웹/앱에서는 MCP 연결을 지원하지 않아 베타 가이드에서는 제외했습니다.

**피드백 수집**
- 사용 패턴 (어떤 도구를 많이 쓰는지)
- 불편 사항 / 개선 요청
- 정식 출시 시 유료 전환 의향

### 1-4. 사용자 경험

```
① 사용자가 개인 API 키를 이메일로 수령
② 연동 가이드 페이지 접속
③ 사용 중인 AI 플랫폼(Claude, ChatGPT 등)의 MCP 설정에 서버 URL + API 키 입력
④ 연결 완료 → AI에게 "카카오톡 MAU 알려줘" 같은 자연어 질문으로 데이터 조회
```

## 3. MCP 서버 현재 상태

### 3-1. 현재 구현 완료 항목

| 항목 | 상태 | 설명 |
|------|------|------|
| 도구 27개 | ✅ | Insight API 엔드포인트 1:1 매핑 (앱 검색, 사용량, 경쟁사 비교, 이탈, 페르소나 등) |
| 도구 이름/설명 | ✅ | AI가 도구를 자동 선택할 수 있도록 한국어 설명 포함 |
| 파라미터 설계 | ✅ | 필수/선택 구분, 날짜 형식(yyyymmdd) 명시 |
| API 키 인증 | ✅ | Bearer 토큰 검증. 허용 키 목록 기반 (환경변수) |
| SSE 통신 | ✅ | 원격 접속 가능한 HTTP 서버 방식 |
| 헬스체크 | ✅ | `GET /health` 엔드포인트 내장 |
| 에러 처리 | ✅ (기본) | API 에러, 인증 실패 시 에러 메시지 반환 |

### 3-2. 정식 출시 시 보완 필요 항목

MCP 서버는 데이터를 전달하는 역할입니다. 아래 항목은 MCP 서버에서 제어할 수 없습니다.

| 항목 | 설명 |
|------|------|
| AI의 답변 톤/스타일 | AI가 데이터를 받은 후 어떻게 표현할지는 AI 플랫폼(Claude, ChatGPT 등)이 결정 |
| 할루시네이션 완전 차단 | 데이터에 없는 내용을 AI가 추측하여 답변할 가능성. 출처 명시로 최소화하지만 완전 차단은 불가 |
| AI 플랫폼 장애 대응 | AI 플랫폼 자체의 장애/점검은 모바일인덱스에서 대응 불가 |
| 사용자의 질의 품질 | 모호한 질문에 대한 AI의 해석은 플랫폼마다 다를 수 있음 |

그 외 보완 가능한 항목은 정식 출시 버전에서 개발 진행 예정입니다.

**접근 제어**

| 항목 | 현재 | 정식 출시 시 |
|------|------|------------|
| 무료/유료 구분 | 없음 (전체 도구 제공) | 무료는 기본 도구만, 심화 도구는 유료 전용 |
| 호출 제한 | 없음 | 분당/일일 호출 제한 (비정상 트래픽 방지) |
| API 키 관리 | 환경변수에 키 목록 수동 관리 | DB 기반 자동 발급/만료/재발급 |

**응답 품질**

| 항목 | 현재 | 정식 출시 시 |
|------|------|------------|
| 데이터 없을 때 | ✅ "현재 데이터 준비 중입니다" 안내 반환 | 조회 가능 기간/범위 등 상세 안내 추가 |
| 데이터 출처 | ✅ 모든 응답에 "모바일인덱스 INSIGHT" 출처 자동 포함 | 기준일, 데이터 갱신 주기 등 상세 출처 포함 |
| 응답 포맷 | API JSON 원본 그대로 전달 | 핵심 수치 요약 + 상세 데이터 병행 |
| 데이터 크기 | 제한 없음 (요청한 만큼 반환) | 기본 반환 개수 제한 (예: TOP 20), 요청 시 확장 |

**운영**

| 항목 | 현재 | 정식 출시 시 |
|------|------|------------|
| 로깅 | 없음 | 호출 도구명, 사용자 키, 응답 시간, 에러 기록 |
| 모니터링 | 없음 | 서버 다운/에러율 급증 시 알림 (Slack 등) |
| 버전 관리 | 수동 | Insight API 변경 시 MCP 도구 자동 동기화 검토 |

## 4. 백엔드 요청 상세

### 4-1. 서버 인프라

| 항목 | 내용 |
|------|------|
| 런타임 | Node.js 18 이상 |
| 포트 | 3001 (또는 지정) |
| 프로토콜 | SSE (Server-Sent Events) — 일반 HTTP 위에서 동작 |
| 상태 | Stateless (DB 연결 없음, 매 요청마다 모바일인덱스 API 호출) |
| 트래픽 | 베타 기준 동시 접속 100명 미만 예상 |

> 아래 배포 옵션은 참고 사항입니다. 개발 시 편의에 따라 판단해주세요.

- EC2 인스턴스 하나에 PM2로 실행
- ECS Fargate 또는 Cloud Run

### 4-2. 도메인 + HTTPS

```
https://mcp.mobileindex.com
```

- AI 플랫폼이 MCP 서버 연결 시 HTTPS를 필수로 요구
- 서브도메인 `mcp.mobileindex.com` 할당 + SSL 인증서 필요

> 아래는 참고 사항입니다. 도메인명은 개발 시 편의에 따라 판단해주세요.

- 베타 단계에서는 임시 도메인도 가능 (예: `mcp-beta.mobileindex.com`)

### 4-3. 환경변수

서버에 설정해야 할 환경변수:

```
MI_API_TOKEN=모바일인덱스_Insight_API_토큰
PORT=3001
ALLOWED_API_KEYS=고객1키,고객2키,고객3키
```

| 변수 | 설명 |
|------|------|
| `MI_API_TOKEN` | 모바일인덱스 Insight API 호출용 토큰 (현재 코드에 하드코딩된 것을 이관) |
| `ALLOWED_API_KEYS` | 베타 고객에게 발급한 키 목록 (쉼표 구분). 새 고객 추가 시 키 추가 후 서버 재시작 |

> 참고: 정식 출시 시에는 DB 기반 키 관리 시스템으로 전환. 베타에서는 환경변수 방식으로 충분합니다.

## Appendix. 백엔드 요청 항목 상세 설명

### ⑥ 서버 인프라

**무엇인가?**
MCP 서버 프로그램을 24시간 돌릴 클라우드 컴퓨터가 필요합니다. 현재는 기획자 노트북에서만 돌아가는 상태라, 외부 사용자가 접속할 수 없습니다.

**왜 기획에서 할 수 없는가?**
AWS 같은 클라우드 서버를 만들고 관리하려면 회사 AWS 계정 접근 권한, 보안 그룹 설정, 서버 모니터링 등이 필요합니다. 인프라 권한이 있는 백엔드 팀만 가능합니다.

### ⑦ 도메인 + HTTPS

**무엇인가?**
사용자가 AI 플랫폼에 입력할 서버 주소(URL)가 필요합니다. `mcp.mobileindex.com` 같은 주소를 만들고, 보안 인증서(HTTPS)를 붙여야 합니다.

**왜 기획에서 할 수 없는가?**
`mobileindex.com` 도메인의 DNS 설정 권한이 필요합니다. 서브도메인 추가와 SSL 인증서 발급은 도메인 관리 권한이 있는 팀만 가능합니다. Claude, ChatGPT 등 AI 플랫폼이 HTTPS가 아니면 연결 자체를 거부하기 때문에 반드시 필요합니다.

### ⑧ 환경변수 설정

**무엇인가?**
MCP 서버가 모바일인덱스 API를 호출하려면 API 토큰이 필요하고, 허가된 사용자만 접속하게 하려면 허용 키 목록이 필요합니다. 이 값들을 서버에 안전하게 설정하는 작업입니다.

**왜 기획에서 할 수 없는가?**
현재 API 토큰이 코드에 직접 박혀 있는데, 이를 서버 환경변수로 옮겨야 보안상 안전합니다. 서버 환경변수 설정은 서버에 직접 접속해야 하므로 서버 접근 권한이 필요합니다.
