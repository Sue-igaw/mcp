# 모바일인덱스 MCP 상품 기획서

## 1. MCP란 무엇인가

### 1-1. 개념

MCP(Model Context Protocol)는 AI가 외부 서비스의 데이터를 직접 가져다 쓸 수 있게 해주는 표준 연결 규격입니다.

쉽게 말하면, **"AI에게 우리 데이터를 읽는 방법을 알려주는 설명서"**입니다.

```
기존 방식:  사용자가 대시보드에 접속 → 데이터를 눈으로 확인 → 복사해서 AI에 붙여넣기
MCP 방식:  AI가 직접 데이터를 가져감 → 사용자는 AI에게 자연어로 질문만 하면 됨
```

### 1-2. MCP 서버를 만들면 어떻게 되는가

MCP 서버 하나를 개발하면, 해당 서버를 지원하는 **모든 AI 플랫폼에서 동시에 사용 가능**합니다.
플랫폼별로 따로 개발할 필요가 없습니다.

```
                    ┌─ Claude (Anthropic)
                    ├─ ChatGPT (OpenAI)
모바일인덱스 MCP ───┼─ Gemini (Google)
    서버 1개        ├─ Copilot (Microsoft)
                    ├─ Kiro (AWS)
                    └─ Cursor, Windsurf 등
```

- 2024년 11월 Anthropic(Claude 개발사)이 오픈소스로 발표
- 2025년 12월 Linux Foundation에 기증 → Anthropic, OpenAI, Google, Microsoft, AWS가 공동 참여하는 업계 공통 표준으로 전환
- 2026년 3월 기준 10,000개 이상의 MCP 서버 운영 중, SDK 월간 다운로드 9,700만 건 이상

### 1-3. 데이터 플랫폼이 MCP 상품을 출시하는 방식

데이터를 보유한 플랫폼이 MCP를 출시하는 방식은 크게 두 가지입니다.

| 방식 | 설명 | 난이도 |
|------|------|--------|
| **정돈된 데이터 API 기반 (권장)** | 이미 운영 중인 API를 MCP 서버로 감싸는 방식. API가 있으면 MCP 서버는 중간 연결 레이어만 추가하면 됨 | 낮음 |
| 데이터베이스 직접 연결 | DB에 직접 쿼리하는 MCP 서버를 개발. API가 없는 경우의 대안이나, 보안/성능 이슈가 큼 | 높음 |

모바일인덱스는 이미 Insight API(27개 엔드포인트)가 운영 중이므로, **가장 수월한 형태(API 기반)로 MCP 출시가 가능**합니다.

### 1-4. MCP로 무엇을 할 수 있는가

사용자가 AI 플랫폼에 모바일인덱스 MCP를 연결하면, 이런 질의가 가능합니다:

| 질의 예시 | AI가 하는 일 |
|----------|------------|
| "카카오톡 MAU 알려줘" | 모바일인덱스에서 카카오톡 MAU 데이터를 자동 조회하여 응답 |
| "증권 업종 앱 순위 TOP 20" | 업종 분류 자동 매핑 → 사용량 순위 조회 → 순위표 생성 |
| "쿠팡과 네이버쇼핑 사용자 비교 분석해줘" | 두 앱의 MAU, 사용시간, 데모그래픽을 각각 조회 → 비교 분석 → 인사이트 도출 |
| "증권 업종 시장 분석 보고서 만들어줘" | 시장 규모 → 경쟁 구도 → 주요 앱 심화 분석 → 이탈 분석 → 전략 제안까지 자동 수행 |

> 📌 모바일인덱스 대시보드에서는 데이터를 "보여주는" 것까지만 가능합니다.
> MCP를 통하면 AI가 데이터를 받아서 **분석, 비교, 인사이트 도출, 보고서 작성까지** 수행합니다.

## 2. 경쟁사 현황

### 2-1. 시밀러웹 (SimilarWeb) — MCP 정식 출시

시밀러웹은 글로벌 웹/앱 트래픽 분석 플랫폼으로, 2025년 9월 MCP 서버를 정식 출시했습니다.

| 항목 | 내용 |
|------|------|
| 출시일 | 2025년 9월 |
| 엔드포인트 | `https://mcp.similarweb.com` |
| 과금 | 별도 MCP 요금 없음. 기존 API 구독(Business $35,000/년~)에 포함. MCP 호출 시 기존 Data Credits 차감 |
| 도구 수 | 23개 (웹사이트 분석 18개 + 검색 분석 2개 + 앱 분석 3개) |
| 지원 플랫폼 | Claude, Cursor, Microsoft Copilot Studio, Zapier, n8n |

**시밀러웹 MCP 도구 구성:**

| 카테고리 | 도구 수 | 주요 내용 |
|----------|--------|----------|
| 웹사이트 분석 | 18개 | 트래픽, 유입 채널, 순위, 오디언스, 인구통계, 광고비, 기술 스택 등 |
| 검색 분석 | 2개 | SERP 순위, 키워드 클릭 추이 |
| 앱 분석 | 3개 | 앱 설치 비율, 앱스토어 순위, 앱 기본 정보 |

시밀러웹이 MCP를 설명하는 핵심 메시지:

> "API가 원시 데이터를 제공한다면, MCP는 AI에게 데이터와 함께 그 데이터를 어떻게 사용해야 하는지 맥락(context)까지 제공한다."

여기서 "맥락"이란, MCP 서버에 정의된 **도구 설명(description)**을 의미합니다.
각 도구에 "이 도구는 언제, 어떤 상황에서 사용해야 하는지"를 자연어로 기술해두면,
AI가 사용자의 질문을 보고 적절한 도구를 자동으로 선택합니다.

이것이 단순 API와 MCP의 핵심 차이이며, MCP 서버 개발 시 가장 공을 들여야 하는 부분입니다.

### 2-2. 그 외 경쟁사 MCP 출시 현황

| 경쟁 대상 | 서비스 | MCP 출시 | 비고 |
|----------|--------|---------|------|
| 모바일인덱스 | 시밀러웹 | ✅ 정식 출시 (2025.09) | 23개 도구. 웹 트래픽 중심, 앱 분석은 3개뿐 |
| 모바일인덱스 | Sensor Tower | ❓ 비공식 | 커뮤니티에서 비공식 MCP 제작 |
| 모바일인덱스 | data.ai | ❌ 미출시 | Sensor Tower에 인수 |
| 디파이너리 | AppsFlyer | ✅ 베타 (2025.07) | 어트리뷰션/캠페인 데이터. 기존 구독에 포함 |
| 디파이너리 | Braze | ✅ 베타 (2025.06) | 38개 도구. CRM/캠페인 데이터. 기존 구독에 포함 |
| 디파이너리 | Adjust / Airbridge / Amplitude | ❌ 미출시 | |
| 트레이딩웍스 | The Trade Desk / Criteo / Google DV360 / Meta Ads | ❌ 미출시 | |

> 📌 MCP를 출시한 경쟁사 3곳(시밀러웹, AppsFlyer, Braze) 모두 MCP를 별도 상품으로 판매하지 않고, **기존 구독에 포함**시키는 방식을 채택하고 있습니다.

### 2-3. 모바일인덱스 MCP 출시 시 시밀러웹과의 차이

| 항목 | 모바일인덱스 MCP | 시밀러웹 MCP |
|------|----------------|-------------|
| 앱 분석 도구 수 | **27개** (사용량, 이탈, 페르소나, 경쟁사 비교, 데모그래픽, 지역 등) | 3개 (설치율, 스토어 순위, 앱 기본 정보) |
| 데이터 깊이 | 앱 사용량(DAU/MAU), 사용시간, 이탈/유입, 페르소나, 관심 업종 등 심화 데이터 | 웹 트래픽 중심. 앱은 설치율/순위/기본 정보만 |
| 한국 시장 | 한국 앱 시장 데이터 정밀도 우위 | 글로벌 커버리지 중심 |
| 웹 분석 | 미제공 | 18개 도구 (트래픽, 채널, 오디언스 등) |

> 📌 **모바일 앱 심화 데이터(사용량, 이탈, 페르소나, 경쟁사 비교 등)를 MCP로 제공하는 서비스는 현재 시장에 없습니다.**

## 3. 전사 확장 가능성

모바일인덱스뿐 아니라, 디파이너리와 트레이딩웍스도 MCP 형태로 개발된다면,
AI 플랫폼 하나에서 풀 퍼널 데이터를 끊김 없이 활용할 수 있습니다.

```
시장은 어떤가         →  광고를 어떻게 집행하나   →  고객이 어떻게 반응하나
(모바일인덱스 MCP)       (트레이딩웍스 MCP)          (디파이너리 MCP)
```

| 서비스 | MCP 역할 | 질의 예시 |
|--------|---------|----------|
| 모바일인덱스 | 시장/경쟁사 데이터 | "증권 업종 Top 10 앱 MAU 추이", "경쟁앱 사용자 이탈 분석" |
| 트레이딩웍스 | 광고 집행 데이터 | "매체별 ROAS 비교", "CTR 낮은 소재 찾아줘" |
| 디파이너리 | 고객 행동 데이터 | "리텐션 캠페인 성과", "이탈 위험 세그먼트" |

AI 플랫폼은 여러 MCP 서버를 동시에 연결할 수 있으므로, 사용자가 하나의 대화에서:

```
"우리 앱의 시장 포지션, 광고 효율, 고객 리텐션을 종합 분석해줘"

→ 모바일인덱스 MCP: 업종 내 순위, 경쟁앱 대비 MAU/점유율
→ 트레이딩웍스 MCP: 매체별 광고 성과, 예산 효율
→ 디파이너리 MCP: 유입 경로별 리텐션, 이탈 위험 세그먼트
→ AI가 3개 데이터를 종합하여 풀 퍼널 인사이트 도출
```

이런 조합이 가능한 회사는 시장에서 IGAW뿐입니다.

> 📌 모바일인덱스 MCP에서 서버 설계 패턴(도구 설계, 응답 포맷, 인증 구조)을 확립하면,
> 트레이딩웍스/디파이너리는 같은 패턴에 데이터만 바꿔 넣으면 됩니다.
> 모바일인덱스 MCP가 전사 MCP 전략의 파일럿이자 설계 표준이 됩니다.

## 4. 모바일인덱스로 MCP를 시작하자

### 4-1. 모바일인덱스가 MCP 첫 번째 주자로 적합한 이유

모바일인덱스는 전사 서비스 중 MCP 출시 조건이 가장 잘 갖춰져 있습니다.

| 조건 | 모바일인덱스 현황 |
|------|-----------------|
| 정돈된 API | ✅ Insight API 27개 엔드포인트 이미 운영 중 |
| 정형화된 데이터 | ✅ 수치 기반 통계 데이터 → AI가 잘못 해석할 가능성 낮음 |
| 추가 개발 범위 | MCP 서버(중간 연결 레이어)만 추가하면 됨. 새로운 데이터 파이프라인 불필요 |

자체 AI 기능을 탑재하는 것과 비교하면:

| 항목 | MCP (AI에 데이터를 제공) | 자체 AI 기능 (서비스에 AI를 탑재) |
|------|----------------------|-------------------------------|
| 개발 범위 | MCP 서버 (중간 레이어) | AI 모델 연동 + 채팅 UI + 프롬프트 설계 등 |
| 개발 기간 | 6~8주 | 12~16주 이상 |
| 도달 범위 | Claude/ChatGPT/Gemini 전체 사용자 | 모바일인덱스 기존 고객만 |
| AI 품질 | Claude, GPT 등 최고 수준 AI 활용 | 자체 구현 수준에 의존 |
| 비용 | 서버 운영비만 | AI API 호출 비용 + 개발비 |

> 📌 사용자가 이미 쓰고 있는 AI 플랫폼에 MCP를 붙이는 것이, 모바일인덱스에 AI를 탑재하는 것보다 훨씬 자연스럽고 수월합니다.

### 4-2. 매출 폭등을 기대하는 것이 아니다

MCP 출시의 목적은 단기 매출 증대가 아닙니다.

**디지털 인터페이스의 전환에 대응하는 것입니다.**

현재 데이터 서비스의 인터페이스가 전환되고 있습니다:

```
웹/앱 대시보드 (직접 조작)  →  AI 대화형 인터페이스 (자연어 질의)
```

- 한국 직장인의 **63.5%**가 생성형 AI를 사용해본 경험이 있으며, 업무 목적 사용자는 **51.8%**
- 한국의 업무용 AI 도입률은 미국(26.5%)의 약 2배
- AI 사용으로 주당 평균 1.5시간 절약
- 전문직(69.2%), 관리직(65.4%), 사무직(63.1%)에서 특히 높은 사용률

> ([출처: 한국은행 "AI의 빠른 확산과 생산성 효과" 보고서, 2025.08](https://www.khan.co.kr/article/202508191706357/?eng))

모바일인덱스 고객(마케터, 기획자, 리서처, 경영진)은 이미 AI를 업무에 활용하고 있습니다.
이들이 데이터를 소비하는 방식이 "대시보드 접속"에서 "AI에게 질문"으로 전환되고 있는 상황에서,
**우리도 이 흐름에 빠르게 편승해야 합니다.**

경쟁사 시밀러웹은 이미 2025년 9월에 MCP를 출시했습니다.

MCP 출시를 통해 기대하는 것:

| 기대 효과 | 설명 |
|----------|------|
| 사용자 경험 전환점 | 대시보드 → AI 대화형으로의 혁신적 사용 경험 제공 |
| 기존 고객 락인 | 프리미엄 고객 한정 기능으로 제공 시, 에센셜 고객의 버전업 또는 기존 고객 이탈 방지 효과 |
| 심화 데이터 활용 증가 | MAU만 확인하던 고객이 이탈/페르소나/경쟁사 비교 등 심화 데이터까지 자연스럽게 사용 |
| 경쟁사 대응 | 시밀러웹 MCP 대비 앱 심화 데이터 차별화 |
| 전사 MCP 전략의 파일럿 | 모바일인덱스에서 설계 패턴을 확립하여 트레이딩웍스/디파이너리 확장의 기반 마련 |

### 4-3. 현재 워크플로우의 문제와 MCP가 바꾸는 것

**현재 사용자의 워크플로우:**

```
① 시장 트렌드 파악     → 웹 서치 (지금 트렌드가 어떻지?)
② 데이터 수집          → 모바일인덱스 대시보드 (MAU 얼마지?)
③ 데이터 변화 원인 분석 → 다시 웹 서치 (왜 올랐지? 왜 떨어졌지?)
④ 보고서 작성          → PPT, 워드, 엑셀 (차트 만들고, 인사이트 쓰고)
```

이 과정에서 데이터 수집 + 정리에 전체 작업 시간의 60~70%가 소모됩니다.
모바일인덱스에서는 데이터를 "보여주는" 것까지만 가능하고, 분석과 인사이트 도출은 전부 사람의 몫입니다.

**MCP가 바꾸는 워크플로우:**

```
사용자: "증권 업종 앱 시장 분석 보고서 만들어줘"

AI가 하나의 대화 안에서 연속 실행:
  ① 업종 분류 자동 매핑
  ② 업종 전체 MAU 추이 조회
  ③ Top 앱 순위 + 점유율
  ④ 주요 앱 상세 비교
  ⑤ 이탈/유입 분석
  ⑥ 데이터 종합 → 인사이트 도출 → 보고서 작성
```

| 기존 | MCP 이후 |
|------|---------|
| 도구 4개를 왔다갔다 (웹 → 모바일인덱스 → 엑셀 → PPT) | AI 플랫폼 하나 |
| 데이터 수집에 60~70% 시간 | 데이터 수집은 AI가 자동. 사용자는 인사이트에 집중 |
| 경쟁앱 5개 분석 = 같은 조작 5번 반복 | "경쟁앱 5개 비교해줘" 한 마디 |
| 모바일인덱스는 데이터만 보여줌 | AI가 데이터를 받아서 분석 + 인사이트 + 보고서까지 |

### 4-4. 사용자에 따라 목적도, 도구도 달라진다

MCP로 데이터를 제공하는 것의 가장 큰 핵심은, **같은 데이터라도 사용자의 목적에 따라 완전히 다른 도구로 변환된다**는 것입니다.

MCP는 하나의 상품이지만, 사용자가 어떻게 쓰느냐에 따라 전혀 다른 가치를 만들어냅니다:

| 사용자 | MCP가 되는 것 | 사용 예시 |
|--------|-------------|----------|
| 리서치/대행사 | **시장 분석 도구** | "핀테크 업종 전체 시장 규모와 Top 20 앱 경쟁 구도를 분석해서 리포트 만들어줘" → AI가 업종 데이터 수집 + 경쟁 분석 + 리포트 초안까지 자동 생성 |
| 앱 마케터 | **경쟁사 인텔리전스 도구** | "우리 앱에서 이탈한 사용자가 어디로 갔는지, 경쟁앱 대비 어떤 연령대가 약한지 분석해줘" → 이탈/페르소나/데모그래픽 심화 데이터를 AI가 자동 조합 |
| 경영진/리드 | **성과 보고서 자동화 도구** | "이번 달 우리 앱 시장 포지션 변화와 경쟁사 동향을 경영진 보고 양식으로 정리해줘" → 매월 반복되는 보고서를 AI가 자동 생성 |
| 연구원/애널리스트 | **데이터 수집 + 분석 어시스턴트** | "증권 업종 Top 10 앱의 12개월 MAU, 사용시간, 이탈률을 모두 수집해서 비교표 만들어줘" → 수십 건의 데이터 수집을 AI가 수분 내 처리 |

여기서 더 나아가, 자동화 도구(n8n, Zapier 등)와 결합하면 **사용자의 자율성이 한 단계 더 높아집니다.**

| 심화 활용 | 설명 |
|----------|------|
| 주간/월간 보고서 자동 발송 | 사용자가 정한 양식으로 매주/매월 AI가 데이터 수집 → 리포트 생성 → 이메일/Slack 자동 전달 |
| 경쟁앱 모니터링 알림 | 경쟁앱 MAU 급등, 우리 앱 순위 하락 등 조건 충족 시 Slack/이메일 알림 |
| 커스텀 대시보드 자동 업데이트 | 관심 앱/지표를 매일 자동 수집하여 Google Sheets/Notion에 기록 |

> 📌 실제 사례: AppsFlyer MCP 사용자가 n8n과 결합하여 10개 클라이언트의 주간 UA 리포트를 자동화.
> 설정 시간 15분, 코딩 불필요. 매주 수시간 걸리던 작업이 완전 자동화됨.
> ([출처: gamigion.com](https://www.gamigion.com/stop-doing-ua-reports-i-automated-my-weekly-ua-reporting/))

모바일인덱스가 "기능"을 만들어주는 것이 아닙니다.
데이터를 AI에 연결해주면, **사용자가 자기 업무에 맞게 자율적으로 활용**합니다.
자동화 도구와의 결합까지 포함하면, 사용자가 커스터마이즈할 수 있는 범위는 매우 넓어집니다.

### 5-1. 어디에 — 지원 AI 플랫폼

| ChatGPT (OpenAI) | ✅ 공식 지원 | 🥇 1순위 | 사용자 풀 가장 큼. 한국 직장인 AI 사용의 대부분 |

### 5-2. 어떻게 — 과금 구조

기존 API 구독에 MCP 접근을 추가하는 방식을 권장합니다.

```
기존 API 구독 고객 → MCP 접근 포함 (추가 비용 없음)
신규 고객         → MCP 포함 API 구독 상품 가입
```

경쟁사 3곳(시밀러웹, AppsFlyer, Braze) 모두 이 방식을 채택하고 있으며,
MCP 자체로 추가 매출을 만들기보다 기존 구독의 가치를 높여 이탈 방지 + 업셀링을 유도하는 방향입니다.

**무료 버전(Free 티어):**

유료 전환 유도를 위해 제한된 무료 체험을 제공합니다.

| 기능 | Free | 유료 (구독 고객) |
|------|------|----------------|
| 앱 검색 | ✅ | ✅ |
| 업종 대분류/소분류 조회 | ✅ | ✅ |
| 통합 사용자 수 순위 Top 10 | ✅ | ✅ |
| 특정 앱 상세 분석 (사용량, 데모그래픽) | ❌ | ✅ |
| 경쟁앱 비교 분석 | ❌ | ✅ |
| 이탈/페르소나/지역 분석 | ❌ | ✅ |
| 조회 기간 | 최근 1개월만 | 전체 기간 |
| 월 호출 제한 | n회 | 무제한 |

> 📌 무료 사용자가 심화 도구를 호출하면 "이 기능은 유료 구독에서 사용 가능합니다" 메시지를 반환하여 업셀링을 유도합니다.

## 6. 향후 확장 가능성

MCP 출시 이후, 단계적으로 확장할 수 있는 방향입니다.

### 6-1. MCP 전용 데이터 — 대시보드에서 볼 수 없는 새로운 가치

현재 Insight API 27개 도구 기반으로 출시하되, 향후 MCP 전용 데이터를 추가 개발하면
**기존 모바일인덱스 대시보드에서는 제공하지 않는 새로운 가치**를 만들 수 있습니다.

| 확장 방향 | 설명 | 기대 효과 |
|----------|------|----------|
| 복합 분석 도구 | 여러 API를 묶어 "경쟁앱 종합 분석", "업종 리포트" 같은 원스텝 도구 제공 | 사용자가 도구를 조합할 필요 없이 한 번에 결과 획득 |
| 크로스 데이터 분석 | 사용량 + 이탈 + 페르소나를 교차 분석하는 전용 도구 | "이탈 사용자의 페르소나는?" 같은 복합 질의 지원 |
| 트렌드 알림 데이터 | 업종별 급상승/급하락 앱, 신규 진입 앱 등 변화 감지 데이터 | 시장 변화를 능동적으로 포착 |
| 벤치마크 데이터 | 업종 평균 대비 특정 앱의 위치를 자동 산출 | "우리 앱 MAU가 업종 평균 대비 어떤 수준인지" 즉시 답변 |

> 📌 이러한 MCP 전용 데이터/도구는 대시보드에서는 제공하지 않는 것으로,
> MCP 사용자만의 차별화된 가치가 되며, 유료 전환의 근거가 됩니다.

### 6-2. 자사 추가 개발 가능한 도구/서비스

| 확장 도구 | 설명 | 개발 범위 |
|----------|------|----------|
| 프롬프트 템플릿 라이브러리 | "경쟁앱 분석", "업종 리포트", "월간 보고서" 등 바로 쓸 수 있는 질의 템플릿 제공 | 낮음 |
| 자동화 워크플로우 템플릿 | n8n/Zapier용 사전 제작 워크플로우를 제공하여 사용자가 복사 → 바로 사용 | 낮음 |
| 모바일인덱스 Custom GPT | ChatGPT 무료 사용자도 접근 가능한 모바일인덱스 전용 GPT (AppsFlyer 사례 참고) | 낮음 |
| 정기 리포트 서비스 | 모바일인덱스가 직접 운영하는 자동 리포트 발송 서비스 | 중간 |
| AI 마켓플레이스 등록 | Claude 커넥터 디렉토리, MCP 커뮤니티(mcp.so, Smithery) 등에 등록하여 신규 유입 | 낮음 |

## 7. 어떻게 사용자에게 다가갈 것인가

### 7-1. AI 플랫폼이 빵 뜬 계기를 활용한다

ChatGPT가 2022년 말 출시 후 폭발적으로 성장한 계기를 돌아보면, 사람들이 AI를 쓰기 시작한 건 "AI가 좋다"는 설명을 들어서가 아닙니다. **누군가가 만든 결과물을 보고 "이게 된다고?"라고 놀랐기 때문**입니다.

- SNS에서 AI가 쓴 글, AI가 그린 그림, AI가 짠 코드가 퍼지면서 → "나도 해볼까?"
- 직접 써보니 생각보다 쓸만해서 → 업무에 적용
- 동료가 쓰는 걸 보고 → "나만 안 쓰면 뒤처지겠다"

이 패턴을 그대로 활용합니다.

### 7-2. 핵심 전략: 결과물 먼저, 도구 설명은 나중에

MCP를 설명하는 것이 아니라, **MCP로 만든 결과물을 먼저 보여주고** "이거 어떻게 만들었냐면요"로 연결합니다.

```
❌ "MCP라는 기술이 있는데, 이걸 쓰면 AI에서 데이터를 조회할 수 있습니다"
✅ "이 증권 업종 분석 보고서, AI한테 한 마디 했더니 3분 만에 나왔습니다. 어떻게요? 모바일인덱스 MCP입니다."
```

| 단계 | 방법 | 설명 |
|------|------|------|
| ① 결과물 공개 | 블로그, SNS, 뉴스레터 | MCP로 생성한 업종 분석 리포트, 경쟁앱 비교 차트, 전략 보고서 등을 콘텐츠로 공개 |
| ② 관심 유발 | "이거 어떻게 만들었어요?" | 결과물의 품질에 놀란 사용자가 방법을 궁금해함 |
| ③ MCP 소개 | "모바일인덱스 MCP로 만들었습니다" | 결과물 → 도구 순서로 자연스럽게 MCP를 인지 |
| ④ 전환 유도 | "지금 바로 써보세요" | 무료 체험 / 베타 참여 / 구독 안내 |

### 7-3. 맛보기 데이터로 진입 장벽을 없앤다

AI 플랫폼도 처음엔 다들 낯설어서 안 쓰려고 했습니다. MCP도 마찬가지입니다.
"일단 써보게 만드는 것"이 가장 중요합니다.

| 전략 | 설명 |
|------|------|
| 무료 맛보기 데이터 | 앱 검색, 업종 분류, Top 10 순위 등 기본 데이터를 무료로 제공. 가입만 하면 바로 체험 가능 |
| 5분 연결 가이드 | "5분이면 AI에 연결됩니다" — 설정 과정이 간단하다는 것을 강조 |
| 프롬프트 복사 붙여넣기 | "이 질문을 그대로 복사해서 AI에 붙여넣어 보세요" — 뭘 물어봐야 할지 모르는 문제 해결 |
| 결과물 미리보기 | 무료 사용자가 심화 도구를 호출하면 "이런 결과가 나옵니다" 샘플을 보여주고 유료 전환 유도 |

### 7-4. 함께 배포할 콘텐츠

| 콘텐츠 | 설명 | 목적 |
|--------|------|------|
| 연동 가이드 | AI 플랫폼별 MCP 연결 방법 (5분 이내) | 진입 장벽 제거 |
| 프롬프트 가이드 | "이렇게 물어보면 이런 결과가 나옵니다" 예시 모음 | 사용법 안내 |
| 시나리오 가이드 | 직군별 활용 시나리오 (리서처용, 마케터용, 경영진용) | 자기 업무에 어떻게 쓸지 구체적으로 안내 |
| 자동화 가이드 | n8n/Zapier 연동으로 주간 보고서 자동화, 알림 설정 방법 | 파워 유저 전환 |
| 데모 영상 | 실제 AI에서 MCP로 데이터 조회 → 보고서 생성하는 과정 녹화 | 시각적 체감 |

### 7-5. 사용자 확보 채널

| 채널 | 방법 | 기대 효과 |
|------|------|----------|
| 기존 고객 (AM) | AM 팀이 결과물 + 가이드를 직접 전달 | 기존 고객 전환, 프리미엄 업그레이드 유도 |
| 모바일인덱스 사이트 | 상품안내에 MCP 섹션 추가 + 결과물 샘플 노출 | 사이트 방문자 대상 인지도 확보 |
| 콘텐츠 마케팅 | MCP로 만든 리포트를 블로그/SNS에 공개 | "이런 것도 되는구나" 인식 확산, 바이럴 유도 |
| 베타 테스터 모집 | "MCP 베타 테스터 모집" 캠페인 | 사전 홍보 + 얼리어답터 확보 |
| 외부 커뮤니티 | 마케팅/데이터 분석 커뮤니티, 업계 미디어 | 신규 잠재 고객 유입 |
| AI 마켓플레이스 | Claude 커넥터 디렉토리, MCP 서버 디렉토리(mcp.so 등) 등록 | AI 플랫폼 사용자 자연 유입 |

### 7-6. 지속 사용 유도

| 전략 | 설명 |
|------|------|
| 온보딩 시나리오 | 첫 연결 후 "이 3가지를 먼저 해보세요" (내 앱 MAU 확인 → 경쟁앱 비교 → 업종 리포트) |
| 정기 활용 팁 | 주간/월간으로 "이번 주 이렇게 써보세요" 이메일 발송 |
| 자동화 설정 유도 | 한 번 자동화를 설정하면 매주 자동으로 사용하는 구조 → 이탈 방지 |
| 무료 → 유료 전환 | 심화 도구 호출 시 "이 기능은 유료에서 사용 가능합니다" + 샘플 결과 미리보기 |

## 8. NEXT PLAN — 베타 출시

### 8-1. 베타를 먼저 돌려야 하는 이유

위에서 제시한 기대 효과, 사용 시나리오, 홍보 전략은 모두 **가설**입니다.
이 가설이 맞는지는 실제 사용자가 써봐야만 답이 나옵니다.

검증해야 할 것:
- MCP가 실제로 고객의 업무 효율을 높이는가?
- 고객이 가장 많이 조회하는 데이터는 무엇인가? (MAU? 이탈? 페르소나?)
- 심화 데이터 활용이 실제로 늘어나는가?
- 추가로 필요한 기능이나 데이터가 있는가?

현재 기획 측 개발(MCP 서버 27개 도구, SSE 통신, API 키 인증, 연동 가이드)은 **이미 완료**된 상태입니다.
기존 Insight API를 기반으로 최대한 빠르게 베타 버전을 배포하고,
실제 고객의 피드백을 받아 정식 출시 방향을 확정하는 것을 제안합니다.

```
지금 제안하는 방향:
  베타 배포 → 실사용 피드백 수집 → 반응 좋으면 고도화 + 정식 출시

피하고 싶은 방향:
  가설만으로 정식 출시 설계 → 방향이 맞는지 모른 채 개발 리소스 투입
```

### 8-2. 베타 테스트 설계

**대상:**
- 기존 고객사 중 베타 참여 의향이 있는 곳 (AM 팀 섭외)
- 내부 보안 이슈가 없는 고객사 우선
- 현재 AI 플랫폼(Claude, ChatGPT 등)을 사용 중인 담당자가 있는 곳

**제공 범위:**
- MCP 서버 전체 27개 도구 (Insight API 전체 기능)
- 지원 AI 플랫폼: Claude, ChatGPT, Kiro, Cursor, Windsurf

**사용자 경험:**
```
① API 키를 이메일로 수령
② 연동 가이드 페이지 접속 (https://mi-mcp-beta-guide.web.app)
③ 사용 중인 AI 플랫폼에 서버 URL + API 키 입력 (5분)
④ AI에게 "카카오톡 MAU 알려줘" 같은 자연어 질문으로 데이터 조회
```

### 8-3. 피드백 수집 항목

| 항목 | 목적 |
|------|------|
| 실제 업무 효율 변화 | MCP가 기존 대시보드 대비 시간을 절약해주는가 |
| 가장 많이 사용하는 도구/데이터 | 어떤 데이터를 중점으로 보는가 |
| 심화 데이터 활용 여부 | 기존에 안 쓰던 심화 데이터를 MCP를 통해 사용하게 되었는가 |
| 추가 필요 기능 | 현재 27개 도구 외에 추가로 필요한 데이터나 기능이 있는가 |
| 불편 사항 | 연결 과정, 응답 품질, 에러 등에서 개선이 필요한 부분 |
| 유료 전환 의향 | 정식 출시 시 유료로 사용할 의향이 있는가 |

### 8-4. 베타 모집 — 홍보 효과를 함께 얻는다

베타 테스트 모집 자체가 MCP 상품의 사전 홍보 역할을 합니다.

| 채널 | 방법 | 기대 효과 |
|------|------|----------|
| 기존 고객 (AM 섭외) | AM 팀이 기존 고객사에 베타 참여 안내 | 프리미엄 고객 락인, 업그레이드 유도 |
| 모바일인덱스 사이트/뉴스레터 | "MCP 베타 테스터 모집" 배너 또는 메일 발송 | 기존 고객 대상 MCP 인지도 확보 |
| 외부 채널 (선택) | 마케팅/데이터 커뮤니티, SNS 등에 베타 모집 공지 | 신규 잠재 고객 유입, 모바일인덱스 브랜드 노출 |

| 부가 효과 | 설명 |
|----------|------|
| 사전 홍보 | "모바일인덱스가 AI 연동 상품을 준비하고 있다"는 시장 인지 형성 |
| 얼리어답터 확보 | 베타 참여자가 정식 출시 시 첫 유료 고객이 될 가능성 높음 |
| 콘텐츠 소재 | 베타 사용자의 활용 사례를 정식 출시 시 마케팅 콘텐츠로 활용 가능 |
| 경쟁사 대응 시그널 | 시밀러웹 MCP 대비 "우리도 준비하고 있다"는 시장 메시지 |

### 8-5. 베타 이후 — 피드백에 따라 방향 결정

```
✅ 반응이 좋다면:
   → 피드백 반영하여 도구 품질 고도화
   → 프롬프트 템플릿 / 자동화 가이드 추가
   → 과금 체계 확정 후 정식 출시

⚠️ 특정 데이터에 수요가 집중된다면:
   → 해당 데이터 중심으로 도구 고도화
   → 불필요한 도구는 정리하여 AI의 도구 선택 정확도 향상

📝 추가 기능 요청이 있다면:
   → 기존 API에 없는 데이터는 백엔드 팀과 협의하여 API 확장 검토
   → 복합 도구(여러 API를 묶은 시나리오 도구) 추가 개발
```

### 8-6. 베타 출시에 필요한 잔여 작업

기획 측 개발은 완료된 상태이며, 백엔드 인프라 작업이 완료되면 베타 출시가 가능합니다.

| 항목 | 담당 | 설명 | 상태 |
|------|------|------|------|
| MCP 서버 코드 | 기획 | Insight API 27개 도구 구현 | ✅ 완료 |
| SSE 통신 (원격 접속) | 기획 | 로컬 → 인터넷 접속 가능하게 변환 | ✅ 완료 |
| API 키 인증 | 기획 | Bearer 토큰 검증 | ✅ 완료 |
| 연동 가이드 웹페이지 | 기획 | 5개 AI 플랫폼별 설정 가이드 | ✅ 완료 |
| 가이드 배포 | 기획 | https://mi-mcp-beta-guide.web.app | ✅ 완료 |
| 서버 인프라 | 백엔드 | 클라우드 서버 세팅 (EC2 하나면 충분) | ❌ 대기 |
| 도메인 + HTTPS | 백엔드 | `mcp.mobileindex.com` + SSL 인증서 | ❌ 대기 |
| 환경변수 + 배포 | 백엔드 | API 토큰, 허용 키 목록 설정 후 서버 실행 | ❌ 대기 |
| 연동 테스트 | 합동 | AI 플랫폼 연결 테스트 + 서버 모니터링 | ❌ 대기 |

> 📌 상세 개발 요청 사항은 별도 문서(MCP 베타 개발 요청서)에 정리되어 있습니다.
