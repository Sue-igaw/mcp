1. 개요

1-1. 테스트 현황

💡 사용하는 LLM 및 에이전트에 따라 답변 표현이 다를 수 있으나, 데이터 조회 결과 자체는 동일하며 큰 차이는 없을 것으로 예상

플랫폼형태연결 결과BETA 대상설명
KiroIDE🟢 성공✅ 대상AWS AI IDE, MCP 네이티브 지원
Kiro CLICLI— 미지원❌ 비대상현재 Kiro CLI에서 MCP 미지원
Claude DesktopDesktop🟢 성공✅ 대상Anthropic 데스크탑 앱, MCP 최초 개발사
Claude CLICLI🟢 성공✅ 대상SSE 원격 서버 연결 가능, Desktop과 동일한 방법

1-2. 목표

  • 모바일인덱스 MCP 서버 원격 배포, 가이드에 따라 API 키 + 서버 URL만 입력하면 AI에서 모바일인덱스 데이터를 자연어로 조회
  • 내부 프로토타입 테스트를 통해 실제 사용 패턴 확보 및 개선점 도출
  • 테스트 결과를 바탕으로 MCP 서비스의 목적 및 방향성, 정식 출시 진행 여부를 판단

1-3. 운영 계획

항목내용
대상아이지에이웍스 내부 필요 부서 (마케팅팀, AM팀 등, 외부 고객사는 대상 ❌)
제공 범위MCP 서버 전체 27개 도구 (모바일인덱스 INSIGHT API 전체 기능) + Small DMP* DB 연동
지원 AI 플랫폼: KiroClaude Desktop
피드백 수집사용 패턴 (어떤 도구를 많이 쓰는지)을 포함한 불편 사항, 개선 요청 외 전반적인 피드백

Small DMP란?
전사 앱 데이터에서 표본을 추출하여 경량화한 DB입니다.
부담이 적은 규모로 구성하여 사용자가 AI(LLM)를 통해 자연어로 원하는 쿼리를 직접 입력하고 결과를 받을 수 있도록 합니다.

1-4. 사용자 경험

순서내용
1️⃣사용자가 개인 API 키를 이메일로 수령
2️⃣연동 가이드 페이지 접속
3️⃣사용 중인 AI 플랫폼(Claude, ChatGPT 등)의 MCP 설정에 서버 URL + API 키 입력
4️⃣연결 완료 → AI에게 "카카오톡 MAU 알려줘" 같은 자연어 질문으로 데이터 조회

1-5. 향후 확장 방향

💡 MCP 상품은 두 가지 데이터 소스로 개발해야 합니다. 빠르게 출시할 수 있는 API 기반부터 먼저 진행합니다.

Insight API 기반 MCP
기존 API로 바로 개발 가능
  • 이미 존재하는 Insight API 27개를 MCP 도구로 변환
  • 모바일인덱스에서 볼 수 있는 데이터와 동일
  • 기획 측 개발 완료, 서버 배포만 남음
Small DMP 기반 MCP  개발 검토 필요
모바일인덱스에서 볼 수 없는 데이터
  • Small DMP를 만들어 DB 직접 연동
  • 교차 사용자 수 추출 등 심화 분석 가능
  • DMP 구축 일정 협의 필요 (DMP 팀)

3. MCP 서버 현재 상태

3-1. 현재 구현 완료 항목

항목상태설명
도구 27개완료모바일인덱스 API 엔드포인트 1:1 매핑 (앱 검색, 사용량, 경쟁사 비교, 이탈, 페르소나 등)
도구 이름/설명완료AI가 도구를 자동 선택할 수 있도록 한국어 설명 포함
파라미터 설계완료필수/선택 구분, 날짜 형식(yyyymmdd) 명시
API 키 인증완료Bearer 토큰 검증. 허용 키 목록 기반 (환경변수)
SSE 통신완료원격 접속 가능한 HTTP 서버 방식
헬스체크완료GET /health 엔드포인트 내장
에러 처리완료API 에러, 인증 실패 시 에러 메시지 반환

3-2. 정식 출시 시 보완 필요 항목

💡 MCP 서버는 데이터를 전달하는 역할이기 때문에, MCP 서버에서 제어할 수 없는 영역 존재

💡 그 외 보완 가능한 항목은 정식 출시 버전에서 진행 예정

🔴 제어할 수 없는 영역

항목설명
AI의 답변 톤/스타일AI가 데이터를 받은 후 어떻게 표현할지는 AI 플랫폼(Claude, ChatGPT 등)이 결정
할루시네이션데이터에 없는 내용을 AI가 추측하여 답변할 가능성, 출처 명시로 최소화하지만 완전 차단은 불가
AI 플랫폼 장애AI 플랫폼 자체의 장애/점검은 모바일인덱스에서 대응 불가
사용자의 질의 품질모호한 질문에 대한 AI의 해석은 플랫폼마다 다를 수 있음

🟢 응답 품질

항목현재정식 출시 버전
데이터 없을 때"현재 데이터 준비 중입니다" 안내 반환조회 가능 기간/범위 등 상세 안내 추가
데이터 출처모든 응답에 "모바일인덱스 INSIGHT" 출처 자동 포함기준일, 데이터 갱신 주기 등 상세 출처 포함
응답 포맷API JSON 원본 그대로 전달핵심 수치 요약 + 상세 데이터 병행
데이터 크기제한 없음 (요청한 만큼 반환)기본 반환 개수 제한 (ex. TOP 20), 요청 시 확장

5. 백엔드 요청 상세

5-1. 서버 인프라

항목내용
런타임Node.js 18 이상
포트3001 (또는 지정)
프로토콜SSE (Server-Sent Events) — 일반 HTTP 위에서 동작
상태Stateless (DB 연결 없음, 매 요청마다 모바일인덱스 API 호출)
트래픽BETA 기준 동시 접속 100명 미만 예상

배포 옵션: EC2 인스턴스 하나에 PM2로 실행 / ECS Fargate 또는 Cloud Run

5-2. 도메인 + HTTPS

https://mcp.mobileindex.com  개발 필요

  • AI 플랫폼이 MCP 서버 연결 시 HTTPS를 필수로 요구
  • 서브도메인 mcp.mobileindex.com 할당 + SSL 인증서 필요
  • BETA 단계에서는 임시 도메인도 가능 (ex. mcp-beta.mobileindex.com)

5-3. 환경변수

💡 정식 출시 단계에서 DB 기반 키 관리 시스템으로 전환, BETA 버전에서는 환경변수 방식으로 진행 가능

MI_API_TOKEN = 모바일인덱스_Insight_API_토큰 PORT = 3001 ALLOWED_API_KEYS = 고객1키, 고객2키, 고객3키
변수설명
MI_API_TOKEN모바일인덱스 Insight API 호출용 토큰 (현재 코드에 하드코딩된 것을 이관)
ALLOWED_API_KEYSBETA 고객에게 발급한 키 목록 (쉼표 구분), 새 고객 추가 시 키 추가 후 서버 재시작

[참고]
PM2 사용 시 : pm2 start mobileindex-mcp-sse.mjs --name mcp-server
헬스체크 (서버 내장) : GET /health → {"status":"ok","tools":27}
